curry (f)

((a, b) -> c) -> a -> b -> c.

Returns a curried equivalent of the provided function. Arguments to curried function needn't be provided on at a time. For example, if f is a ternary function, and local g = curry(f), then the following are equivalent:

  • g(1)(2)(3)
  • g(1, 2)(3)
  • g(1)(2, 3)
  • g(1, 2, 3)

Parameters:

  • f function The function to curry.

Returns:

    function A new, curried function.

See also:

    curryN
curryN (arity, f)

number -> ((a, b) -> c) -> a -> b -> c.

Returns a curried equivalent of the provided function, with a specified arity. Arguments to curried function needn't be provided on at a time. For example, if f is a ternary function, and local g = curryN(3, f), then the following are equivalent:

  • g(1)(2)(3)
  • g(1, 2)(3)
  • g(1)(2, 3)
  • g(1, 2, 3)

Parameters:

  • arity number The arity for the returned function.
  • f function The function to curry.

Returns:

    function A new, curried function.

See also:

    curryN

Table

evolve (transforms, table)
{ s = (a -> a) } -> { s = a } -> { s = a }.

Creates a new table by recursively evolving a shallow copy of table, according to the transform functions. All non-primitive properties are copied by reference.

A transform function will not be invoked if its corresponding key does not exist in the evolved table.

Parameters:

  • transforms table The table specifying transform functions to apply.
  • table table The table to transform.

Returns:

    table A new, transformed table.

Parameters:

  • keys table List of keys to copy onto a new table.
  • table table The table to copy from.

Returns:

    table A new table with only properties from keys on it.
